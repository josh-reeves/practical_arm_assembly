About ARM Assembly

Generally speaking, all computer processors are only capable of understanding a limited set of relatively basic instructions. These instructions typically provide, among other things, basic arithmetic, logical operations and ways to access computer memory. All activity performed by the processor is, at most, an aggregate of these basic operations.

Assembly languages are low-level programming languages that provide a way to directly invoke these instructions. As opposed to higher-level languages (e.g. Python, Rust, the C variants, etc.), which provide controlled and highly-abstracted access to these operations via relatively human-readable code, assembly languages feature a strong correlation between the instructions they provide to the developer and those the processor is actually able to understand. Due to this correlation, assembly languages are architecture specific. This documentation and the associated repository specifically specifically focus on assembly language for ARM processors.

The set of instructions a processor can understand and the the manner in which they are carried out are part of it's ISA (Instruction Set Architecture). The ARM standard includes multiple ISAs. These notes are primarily concerned with the 32-bit A32 ISA and the 64-bit A64 ISA.

Using These Notes

Due to the significant differences between A32 and A64, presenting information for both ISAs simultaneously would negatively impact the clarity and utility of the resulting material. For this reason, the following steps have been taken to organize the included information:

  • Each lesson has been broken into sections. Each of these sections can be collapsed by clicking the section header.
  • Furthermore, many of these sections will include an option to display either 32-bit or 64-bit content. In these situations, the section header will include the buttons shown below. Clicking either button will show content relevant to associated ISA.

Getting Started

The example source code was written and compiled on a Raspberry Pi 5 running Ubuntu Server 24.04. In order to enage with the material to its fullest, a similar setup is recommended. Any 64-bit ARM processor running on a Debian-based Linux distro should be sufficient. The popular system emulator, QEMU can also be used to emulate the necessary architecture.

Once the system is set up, the toolchains required to assemble and link the code will need to be installed. The 32-bit examples were assembled and linked using the gcc-arm-linux-gnueabi toolchain, which can be installed with the following command:

sudo apt-get install gcc-arm-linux-gnueabi -y



The 64-bit examples were assembled and linked with the gcc-aarch64-linux-gnu toolchain, which can be installed with the following command:

sudo apt-get install gcc-aarch64-linux-gnu -y



For simplicity, a docker image has been created that has the above toolchains preinstalled along with the other packages required to compile and run the associated examples and a text editor. For more information, reference the docker file located here. It can be run with the following command, adding any desired volumes or bind mounts as desired:

sudo docker run -dit --name practical-arm-assembly joshreeves/practical-arm-assembly



Once to container is running, the following command can be used to enter its terminal and begin interacting with it:

sudo docker exec -it practical-arm-assembly sh



Alternatively, the CPULator site can be used to execute most of the code from the 32-bit examples. This has the benefit of displaying the values held by various components as the program executes, but some commands cannot be accurately emulated (i.e. The system interruped command). Only the 32-bit examples can be executed this way. The 64-bit examples are not supported.

More detailed instructions for all of the methods described above can be found on the Resources page.
